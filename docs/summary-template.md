# AI 中文摘要模板 v0.2

每篇论文的 `summary.zh.md` 使用以下模板。

```md
---
paper_id: arxiv-2405.12345
summary_version: "0.2"
language: zh
status: ai-draft
---

# 论文中文标题

> 一句话说明论文解决了什么问题、使用什么核心方法，以及最重要的结果。
> 不超过 80 个汉字。

## 研究问题

说明论文试图解决的问题、应用场景，以及此前方法的主要不足。

## 核心方法

用 2—4 个自然段解释方法，重点说明输入、输出、训练方式和推理流程。避免逐段翻译论文或堆砌术语。

## 主要贡献

1. 第一项贡献。
2. 第二项贡献。
3. 第三项贡献。

只写论文能够直接支持的贡献，不把项目宣传语当作事实。

## 数据与训练

说明使用或构建的数据、数据规模、采集方式和训练方式。论文没有提供关键信息时，明确写“论文未披露”。

## 实验与结果

说明：

- 使用的任务、机器人、仿真器或基准；
- 是否包含真实机器人实验；
- 与哪些主要基线比较；
- 最重要的定量或定性结果。

关键结果数字必须标注表格、章节或页码。

## 局限性

总结作者明确提到的限制，以及能够从实验范围直接观察到的限制。推测性判断必须标注为“编辑推断”。

## 为什么值得关注

从 Embodied AI 技术发展角度解释其价值，不重复摘要。

## 适合谁阅读

说明论文适合哪些方向的研究者或工程师。

## 关键资源

- 论文：
- 项目主页：
- 代码：
- 数据：
- 模型：

不存在或没有确认的资源写“未公开”，不要猜测链接。

## 事实核验记录

- [ ] 标题、作者和机构已核验
- [ ] 日期和发表状态已核验
- [ ] 代码与项目链接已核验
- [ ] 实验平台已核验
- [ ] 关键结果数字已核验
- [ ] 真实机器人/仿真结论已核验
```

## 生成规则

- 先读取 `content_type` 和 `source_scope`。只有 `full-source` 才能生成完整长摘要。
- `partial-source` 只能生成待核验初稿；`abstract-and-metadata` 与 `metadata-only` 只能生成快速收录。
- `official-blog` 的方法和结果必须使用“官方博客介绍/报告/声称”等来源限定语，不能写成同行评审结论。
- 对 arXiv 论文，必须基于 ar5iv HTML 全文生成，不能只读 abstract。
- 生成前必须实际读取方法、实验、结论和局限性相关章节，并在 `metadata.json` 中记录 `source_access`。
- ar5iv HTML 无法可靠呈现的图片内容不能视为已核验；依赖图片的判断需通过 PDF 补充核验，否则明确标注不确定性。
- 论文事实优先于项目主页和媒体文章。
- 不得推测未公开的数据规模、训练成本、参数量或机构关系。
- 不得把“计划开源”描述成“已经开源”。
- 不得把仿真结果描述成真实机器人结果。
- 重要数字注明 HTML 章节、表格或图编号，例如“成功率提升 12%（Experiments，Table 2）”；使用 PDF 核验时可补充页码。
- 中文标题属于编辑性翻译，不替代英文原题。
- 建议正文长度为 800—1500 个中文字。
- 除非有充分证据，不自行使用“首次”或“SOTA”；作者自己的声明写成“作者称”。
- 摘要中的事实必须能够追溯到 `metadata.json` 的 `sources`。
