# 20 篇论文试运行计划

## 目标

这轮测试不是追求收录完整度，而是验证：

- 元数据字段能否稳定获得；
- 标签是否能够覆盖不同类型论文；
- AI 摘要能否保持事实准确和结构一致；
- 哪些环节必须人工判断；
- 哪些字段或标签需要在 v1.0 中调整。

## 采样建议

| 类型 | 数量 |
|---|---:|
| VLA / 通用策略 | 4 |
| BC、模仿学习、扩散策略 | 3 |
| RL、规划或控制 | 3 |
| 世界模型 / WAM | 2 |
| 数据集或 Benchmark | 3 |
| 仿真器 | 2 |
| 基础设施或系统 | 1 |
| 综述 | 2 |

总计 20 篇。

## 覆盖要求

- 至少 5 篇多机构合作论文；
- 至少 5 篇包含真实机器人实验；
- 至少 3 篇没有公开代码；
- 至少 3 篇跨越多个板块；
- 新旧论文混合，不只选最近一周的论文；
- 同时包含方法、数据、系统与综述类论文。

## 每篇论文的验收标准

- `metadata.json` 是合法 JSON；
- `schema_version` 为 `0.3`，且具有合法的 `content_type` 与顶层 `source_scope`；
- 正式标签全部来自受控词表；
- 不确定字段没有被猜测补全；
- 已区分首次公开日期与修订日期；
- 已区分真实环境、仿真环境和混合实验；
- 代码、数据和模型链接可访问后才标记为已发布；
- arXiv 论文已成功读取 ar5iv HTML 全文，并记录 `source_access`；
- 已检测到至少三个主要章节，且方法、实验和结论部分已实际读取；
- 摘要基于 HTML 全文并符合统一结构；
- 只能读取摘要页的论文未生成完整长摘要；
- 关键结果数字带有论文定位信息；
- 状态保持为 `ai-draft`。

## 汇总报告

完成 20 篇后生成 `reports/taxonomy-report.md`，至少包含：

1. 每个正式标签的使用次数；
2. 所有 `suggested_new_tags` 及出现次数；
3. 难以分类或跨分类的论文；
4. 无法稳定获得的元数据字段；
5. 容易误判的事实类型；
6. 对 Schema、词表和摘要模板的修改建议；
7. AI 初稿需要人工修改的典型案例。

## v1.0 决策标准

- 从未使用且定义模糊的标签考虑移除；
- 多次出现且无法由现有标签表达的新概念考虑纳入；
- 经常无法核实的字段改为可选或重新定义；
- AI 高频出错的结论增加明确核验规则；
- 只有在 20 篇样本能够一致表达后，才固定 Schema v1.0。
