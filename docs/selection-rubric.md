# Embodied AI Board 候选论文筛选量表 v0.1

## 1. 使用方式

先执行 `discovery-spec.md` 的五项硬门槛。硬门槛全部通过后才进行 100 分评分。每项必须给出一手来源中的简短证据；不得仅凭标题、作者、机构声誉或社交热度给分。

## 2. 评分总览

| 维度 | 满分 | 核心问题 |
|---|---:|---|
| 具身相关性 | 20 | 是否直接推动机器人/具身智能能力或研究基础？ |
| 新颖性与增量 | 20 | 相对最接近工作是否有清楚、重要的新贡献？ |
| 证据质量 | 20 | 实验设计和主张是否匹配、充分且可信？ |
| 潜在影响 | 15 | 是否可能改变能力边界、研究实践或公共基础？ |
| 看板信息增益 | 10 | 是否补充当前看板缺失板块、任务、平台或观点？ |
| 可复现性与资产 | 10 | 方法细节、代码、数据、权重和评测是否支持复核？ |
| 时效性 | 5 | 是否属于本轮真正的新工作或有实质更新？ |
| 合计 | 100 |  |

## 3. 分项锚点

### 3.1 具身相关性（0–20）

- 17–20：核心问题、方法和实验均直接针对机器人或具身智能，真实/仿真物理任务证据充分。
- 12–16：直接服务机器人关键环节，但实验范围较窄，或基础方法与具身应用各占一部分。
- 6–11：具有明确机器人用途，但主要贡献与验证仍在非具身域。
- 0–5：只有泛化叙述或关键词关联，没有直接具身证据；通常不应通过硬门槛。

### 3.2 新颖性与增量（0–20）

- 17–20：提出新的问题设定、能力范式或关键机制，并与最近邻工作做充分对照。
- 12–16：方法组合或系统设计有明确、非平凡增量，消融能定位增益来源。
- 6–11：主要是合理工程整合、规模扩展或有限改进，仍有实用价值。
- 0–5：与已有工作差异模糊、缺少对照，或只更换模型/数据而无新洞见。

### 3.3 证据质量（0–20）

- 17–20：任务/环境多样，基线强，消融与统计充分，主张和实验严格匹配，局限透明。
- 12–16：实验总体可靠，但覆盖、重复次数、基线或统计有一项明显不足。
- 6–11：有支持性结果，但样本有限、对照偏弱或关键主张只被间接验证。
- 0–5：主要结论缺乏可核验证据、比较不公平或仅展示少量案例。

不要因为真实机器人实验天然给高分；仿真工作若规模、控制变量和可复核性更强，也可获得高分。

### 3.4 潜在影响（0–15）

- 13–15：可能显著改变通用能力、数据规模、评测标准或社区基础设施。
- 9–12：对一个重要子方向有较强推进，容易被后续研究采用。
- 5–8：对特定任务、平台或团队实践有明确价值。
- 0–4：影响范围小，且缺少可推广性证据。

### 3.5 看板信息增益（0–10）

- 9–10：填补看板明显空白，或提供与既有论文重要且有证据的相反结论。
- 6–8：扩展已有主题到新任务、平台、模态、数据规模或评测条件。
- 3–5：与已有内容较相似，但仍有一项可区分的新信息。
- 0–2：与看板已有论文高度重复，新增信息有限。

该维度依赖当前 `papers/` 内容，会随看板变化；必须写明比较对象。

### 3.6 可复现性与资产（0–10）

- 9–10：代码、数据/权重、配置和评测协议均公开可访问，方法细节足以复现。
- 6–8：至少有主要代码或数据/权重，正文方法和实验细节较完整。
- 3–5：尚未开放核心资产，但细节充分，或只有部分资产可用。
- 0–2：核心细节和资产均不足，难以独立核验。

只认定当前可访问资产；“即将开源”不计分。闭源本身不等于低质量，但会限制本项得分。

### 3.7 时效性（0–5）

- 5：最近 14 天首次公开。
- 4：最近 15–30 天首次公开。
- 3：旧工作在最近 14 天出现实质修订，新增关键实验、资产或结论。
- 1–2：时间窗外，但因补齐重要主题被人工加入。
- 0：只是重复传播或无实质内容更新。

## 4. 决策阈值

| 总分 | 决策 | 处理方式 |
|---:|---|---|
| 80–100 | `featured` | 建议重点收录；必须人工全文复核后发布。 |
| 65–79 | `include` | 建议常规收录；人工确认关键事实。 |
| 50–64 | `watch` | 简讯、候选池或等待更多证据，不生成正式长摘要。 |
| 0–49 | `reject` | 不收录；保留可审计的拒绝原因。 |

硬门槛失败时，无论估计分数多高，都使用 `reject` 或 `defer`：

- `reject`：明确偏题、重复或证据不足；
- `defer`：全文暂不可访问、关键页面故障或信息尚未公开，之后可重试。

## 5. 防偏差规则

- 机构、作者、会议名称不计分。
- 引用量、点赞、star、转发只作发现信号，不计分。
- 不因“大模型”“通用”“世界模型”等措辞自动提高新颖性或影响分。
- 不因真实机器人演示自动判定证据充分；检查任务数量、试验次数、基线和失败案例。
- 不因开源自动判定研究贡献强；开源主要影响可复现性维度。
- 对同一团队、同一模型系列的连续论文，必须写清相对上一版的实质增量。
- 评分者不确定时降低置信度并列出待核验项，不用主观补全证据。

## 6. 推荐理由模板

```text
这项工作在【问题/方向】上提出【明确增量】，通过【关键实验范围】支持【核心结论】。
相较看板中的【最近邻论文】，新增信息是【差异】。主要风险是【证据限制/未公开资产】。
```

拒绝或暂缓理由必须可操作，例如“全文不可访问，无法核验三项关键实验”，不要只写“质量一般”。
