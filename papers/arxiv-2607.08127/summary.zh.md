# 理解和缓解视频-动作泛化差距：基于时序比率的方法

> 系统研究世界-动作模型（WAM）和视频-动作模型（VAM）在微调后丧失视频基础模型组合泛化能力的现象，提出时序比率（Temporal Ratio）作为可解释诊断工具，并基于此设计推理时自适应引导方法。

## 研究问题

生成式视频基础模型（如 Cosmos-Predict 2.5）在互联网规模视频数据上预训练后展现出强大的组合先验——能理解物体交互、时序连贯性和物理合理性。然而，当这些视频模型被微调为世界-动作模型（WAM）或视频-动作模型（VAM）用于机器人控制时，其组合泛化能力常常严重退化。例如，一个在训练中见过"把奶油奶酪放进碗里"和"把碗放在柜子上"的策略，在测试时面对"把奶油奶酪放在柜子上"这一新组合时仍然失败——尽管涉及的物体、容器和操作原语都已见过。

本文将此现象定义为**视频-动作泛化差距**（Video-Action Generalization Gap, VAG gap），核心研究问题是：视频基础模型与动作预测之间的**正确接口**是什么，才能在微调后保留组合先验？

## 核心方法

### 1. VAM 设计空间系统探索

首先对一类使用潜在视频特征的 VAM（与 Mimic-Video、DiT4DiT 类似）进行了全面的设计空间消融研究，涵盖：

- **微调策略**：LoRA（2-6% 参数）vs 全量微调
- **训练模式**：联合训练（视频损失 + 动作损失）vs 两阶段训练（先微调视频再冻结后训练动作头）
- **视频噪声水平 σᵥ**：从完全噪声（1.0）到完全去噪（0.0）的连续区间
- **预测时间范围 T**：未来帧数

关键发现：标准设计选择无法产生一致的解释模式。ID 和 OOD 性能之间呈现碎片化的差距，没有明确的规律可循。最佳视频噪声水平既不是完全噪声也不是完全去噪，而是在中间值（如 σᵥ=0.883）达到峰值性能。

### 2. 时序比率（Temporal Ratio, TR）

为解释这一碎片化现象，提出**时序比率**——一种基于注意力的度量：

```
TR = 动作 token 对未来帧的注意力 / 动作 token 对当前帧的注意力
```

TR 高表示策略在依赖预测的未来潜在表示进行规划；TR 低表示策略锚定在当前观测上进行精确操作。

TR 的两个关键性质：
1. **预测泛化能力**：TR 与模型的组合泛化能力强相关，可作为 ID-OOD 性能差距的直接预测器
2. **反映任务阶段**：TR 在任务不同阶段自然波动——规划阶段（选择物体、决定放置位置）TR 上升；精确操作阶段（抓取、放置）TR 下降

### 3. TR 自适应引导（TR-Adaptive Guidance）

基于上述发现，提出推理时自适应引导方法：

- **语言引导**：通过对比有条件/无条件视频特征提取，计算语言依赖的动作速度方向
- **规划引导**：使用更长时间范围的视频预测，提取长程时序结构信息
- **动态权重**：引导强度与当前 TR 值成比例缩放——规划阶段大幅增强组合条件信号，操作阶段自动减弱以避免干扰精确控制

## 主要贡献

1. 首次正式定义**视频-动作泛化差距**（VAG gap），系统揭示 WAM/VAM 在微调后丧失视频基础模型组合先验的普遍现象
2. 提出**时序比率**（TR），一个可在策略正常前向传播中在线计算的注意力度量，用于诊断策略何时依赖未来预测进行规划
3. 发现 TR 在两个维度上的可解释性：跨配置预测泛化能力 + 任务阶段内的动态波动
4. 提出**TR 自适应引导**，根据策略当前的 TR 水平动态调节组合条件信号，在规划阶段增强、操作阶段减弱
5. 在 LIBERO 四个组合套件和真实双臂 YAM 任务上大幅超越先前 WAM/VAM

## 数据与训练

**LIBERO 仿真**：使用 LIBERO 四个标准套件（libero-spatial、libero-object、libero-goal、libero-long），每套件 10 个任务，每任务 50 条演示。动作头在成功演示上训练，视频骨干微调使用完整未过滤数据集。

**真实机器人 YAM**：使用 5600 条轨迹、24 个任务的双臂操作数据集，包括盘碟放置、零食分拣、瓶子拾放、拧瓶盖等。三视角 RGB 输入（顶部、左右手腕），输出 14 维双臂动作。

**训练配置**：视频骨干使用 Cosmos-Predict 2.5-2B DiT，动作头使用 300M Gemma Transformer。LoRA 联合训练，最佳 σᵥ=0.883，T=5（17 帧 @10fps），动作头预测 16 动作 @10Hz（仿真）或 50 动作 @30Hz（真实）。

## 实验与结果

### LIBERO 仿真（Table 1）

| 方法 | ID 平均 | OOD 平均 |
|------|---------|----------|
| π₀ (VLA) | 94.2% | 1.7% |
| π₀.₅ (VLA) | 96.9% | 26.8% |
| Cosmos-Policy (WAM) | 98.5% | 10.5% |
| Fast-WAM | 97.6% | 9.4% |
| DiT4DiT (VAM) | 98.6% | 6.4% |
| **本文 (无引导)** | 94.3% | **55.7%** |
| **本文 (含引导)** | 94.0% | **59.4%** |

OOD 性能是先前最佳 WAM/VAM 的 5 倍以上。引导在组合 OOD 套件上提升约 6%，但对 LIBERO-long 略有下降（预测未来不可靠时强行引导反而有害）。

### 真实双臂 YAM（Table 2）

| 方法 | 场景1 | 场景2 | 场景3 | 平均 |
|------|-------|-------|-------|------|
| π₀ | - | - | - | 36.7% |
| π₀.₅ | - | - | - | 55.0% |
| Cosmos-Policy | - | - | - | 12.5% |
| **本文 (无引导)** | 66.7% | 76.7% | 80.0% | **71.7%** |
| **本文 (含引导)** | 76.7% | 90.0% | 90.0% | **83.3%** |

Cosmos-Policy 在多任务双臂数据集上训练失败（训练 1M 步/32×H200），说明大规模多任务 WAM 训练仍是开放问题。提取 σᵥ=1.0 时的特征（如先前 VAM 的做法）在所有 YAM 任务上均失败，强调了部分去噪的结构化视频潜在特征的重要性。

### 关键消融发现

- **联合训练 + LoRA** 产生最强的 OOD 性能
- **两阶段 LoRA** 在不同 σᵥ 和 T 平均下更鲁棒
- **更长视频预测**改善视频级泛化但不一定提升动作成功率——动作头可能忽略正确的组合规划
- 图 4 展示核心失败模式：动作头与视频预测之间的**不一致性**——视频预测对了但动作头不跟，或视频预测错了但动作头仍能执行记忆行为

## 局限性

- 研究限于特定类别的 VAM（潜在视频特征 + 独立动作头），结论可能不适用于联合建模视频和动作 token 的 WAM
- 受限于底层视频模型的物理合理性和组合泛化能力——当生成的未来变得自信但不可行时，自适应引导会放大错误信号
- 额外的去噪步骤和引导增加了推理成本，降低了推理频率
- 代码和模型将在双盲审稿结束后开源
- 仅在 LIBERO 仿真和双臂 YAM 真实任务上验证，场景多样性有限

## 为什么值得关注

本文的核心洞察是：WAM/VAM 的组合泛化能力不取决于视频模型是否预测了正确的未来，而取决于**动作头是否真正使用了这些预测**。时序比率（TR）提供了一个优雅的可解释工具来诊断这一关键问题——它可以从策略的正常前向传播中在线计算，无需额外开销。基于 TR 的自适应引导方法简洁高效，不需要重新训练，在推理时动态放大组合信号。这项工作对于理解从视频基础模型到机器人策略的知识迁移机制具有重要基础研究价值，其"注意力路由"视角也可能启发其他视频条件策略的设计。
