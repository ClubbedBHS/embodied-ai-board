# Neural Task Programming：学习跨层次任务的泛化

> 将 meta-learning 与神经程序归纳相结合，从单次演示中学习可复用的层次化神经程序，在任务长度、拓扑结构和语义目标三个维度上实现泛化。

## 研究问题

机器人操作任务（如物体分拣、积木堆叠、桌面清理）通常具有层次化和组合式结构，涉及长时间的环境交互。传统方法面临两大挑战：(1) 学习能够泛化到**新任务目标**的策略；(2) 将**动作基元层次化组合**以完成长期交互。强化学习依赖任务特定的奖励函数，难以泛化；专家演示的行为克隆则只能复现单一任务。核心问题是：**如何从少量演示中学习能够跨任务泛化的层次化策略？**

## 核心方法

NTP 将任务策略建模为一个**层次化神经程序**（hierarchical neural program），包含三个核心组件：

**1. 任务规范（Task Specification）**：以物体位置轨迹或第三视角视频序列作为输入，描述任务的过程和最终目标。测试时仅需提供**一次演示**（one-shot）作为任务规范。

**2. 程序执行器（Task Specification Interpreter, TSI）**：递归地将任务规范分解为子任务规范。给定当前状态、当前程序和当前规范，TSI 预测：下一层子程序、对应的子任务规范（规范中与子任务相关的连续子序列）、以及是否终止。核心设计是**作用域约束**（scoping constraint）：子任务规范必须是父任务规范的连续子序列，强制信息隐藏，使底层模块仅访问与其功能相关的信息，避免学习虚假依赖。

**3. 反应式核心网络（Reactive Core）**：采用前馈网络（非循环网络）作为核心，使模型对当前环境状态反应灵敏，相比 GRU 等循环结构更能从意外失败中恢复。实验表明在对抗性动力学环境下，NTP 成功率仅下降 20%，而 GRU 变体下降 46%（Section VII-A, Table II）。

底层程序通过**机器人 API**（如 `move_to`、`grip`、`release`）与环境交互，API 参数由 API 特定的解码器预测。高层程序（如 `pick_and_place`）则由这些 API 组合而成。

## 主要贡献

1. 提出了 NTP 框架，将 meta-learning 与神经程序归纳结合，实现了从单次演示中学习可复用的层次化策略。
2. 通过作用域约束强制信息隐藏，使子程序学习模块化语义，支持跨任务的复用和组合。
3. 在三个维度上系统性地验证了泛化能力：任务长度（物体数量增加）、拓扑结构（子任务顺序变化）、语义目标（全新目标配置）。
4. 展示了 NTP 可从视频输入端到端训练（NTPVID E2E），无需显式的物体检测监督。在 Sawyer 机器人上实现了 sim-to-real 迁移。

## 数据与训练

**数据生成**：在 Bullet 物理引擎中以程序方式生成演示轨迹（Section V）。对于 Block Stacking，随机生成 2000 个不同任务实例（不同最终积木配置）；对于 Object Sorting，使用 4 个物体类别和 4 个容器，共 256 种类别-容器映射组合，仅使用 4 个覆盖任务训练，其余 252 个为测试任务。

**训练方式**：监督学习（行为克隆），从演示轨迹中学习。Object Sorting 使用 500 条轨迹训练（Section VI）；Block Stacking 每个任务使用 100 条演示，最多 1000 个训练任务（Section VII）。NTPVID (E2E) 在 1000 个训练任务上使用 8 块 Titan X GPU 训练 10 天（Section VII-B）。Flat 基线使用相同数据量。

## 实验与结果

**仿真实验：**

- **Object Sorting（任务长度）**：NTP 在 4-40 个物体的任务变化中保持 90%+ 成功率，Flat 基线从 40% 降至 25%（接近随机）（Section VI, Figure 5）。NTP (GRU) 在任务变长时性能下降更快。
- **Block Stacking（任务语义 + 拓扑）**：在 1000 个训练任务上，NTP 在 unseen 任务上的表现（~85%）与 seen 任务几乎持平，Flat 基线完全无法泛化到未见任务（~0%）（Section VII, Figure 6）。NTP (no scope) 性能随任务增大逐渐下降，验证了作用域约束的关键作用。
- **Block Stacking（对抗性环境）**：NTP 无故障成功率 0.863，有故障 0.663（↓20%）；NTP (GRU) 无故障 0.884，有故障 0.422（↓46%）（Section VII-A, Table II）。反应式核心更鲁棒。
- **视觉版本**：NTPVID (E2E) 直接从 64×64 图像端到端训练，在 1000 个任务上 success rate 约 0.5-0.6，优于 NTPVID (Detector) 的检测器+规划两阶段方案（Section VII-B, Figure 7）。
- **Table Clean-up**：成功率 55%-100%，随物体数量增加的性能下降主要来自碰撞检查失败，而非 NTP 的决策错误（Section VIII）。

**真实机器人（Sawyer 臂，Table I）：**

| 任务 | 试验次数 | 成功率 | NTP 失败 | 操作失败 |
|------|---------|--------|---------|---------|
| Block Stacking | 20 | 90% | 5% | 5% |
| Object Sorting | 10 | 80% | 0% | 20% |

操作失败包括抓取失败和碰撞检查失败，NTP 本身的算法决策几乎无误。

## 局限性

- 底层 API 为预定义的符号化动作基元，需要人工设计。扩展到更丰富的连续控制接口（速度/力矩控制器）是未来方向。
- 仅支持已知的程序层次结构，无法从数据中自动发现任务分解。
- 任务规范需要人工提供（演示轨迹或视频），未集成语言指令等更灵活的规范方式。
- 缺乏在线失败检测和重规划机制，对感知噪声鲁棒性有限。
- 端到端视频训练（NTPVID E2E）计算成本极高（8 GPU × 10 天），限制了可扩展性。

## 为什么值得关注

NTP 是 2017 年将 **meta-learning** 思想引入机器人操作的先驱工作之一，其核心洞察——将复杂任务分解为可复用的层次化程序、通过信息隐藏实现模块化——在今天的 TAMP 与学习结合的研究中仍然具有启发性。三种泛化维度（长度、拓扑、语义）的系统性评估框架为后续工作提供了基准范式。在 Sawyer 机器人上 80-90% 的真实世界成功率证明了该方法向实际部署的可行性。对于研究**组合式策略学习、分层强化学习、程序归纳**的研究者来说，这是一篇必读的基础文献。
